Application Header / Masthead
Hide Header Header Items are hidden but the header still displays and effects the page layout. The show-app-logo property should be set to false and the header should have no children slots for this variation
Sign In Header Plain white hexagon (no button click). This header variation should be used when the application header doesn't require the heaxagon to be a navigation button. This will usually occur on the sign-in page. In this example, the hexagon is showing a single T "Transform". The T suggests that app, or current state of the app doesn't belong to a specific product.
Navigation Header Gradient hexagon (with button click). The is-logo-clickable property determines the state of the hexagon logo. If it is true then it is rendered as a gradient and emits the custom event tfAppLogoClicked when clicked. The side drawer navigation is expected to listen to tfAppLogoClicked and handle the event itself.
Client Logo Use the component tf-app-client-logo to accept a client logo image and display the image within the fixed image area. The component is added to the header as a child node and uses the clientlogo slot.
Binding properties can be achieved with camelCase property names such as appTitle, appInitials and clientLogoSource
User Account Navigation Within the tf-app-client-logo there is an option to include a logged-in user specific dropdown menu. This is accessed with the property show-app-logo. When this property is true, the component expects the menu options to be passed in through its children Each option in the menu should be a tf-dropdown-option. This works the same way as a dropdown except that this component emits a client logo specific event called tfUserAccountDropdownSelected

Quick Links Similar to the client logo/user account navigation, the header can also accept the application specific quick links dropdown component tf-quick-links. The component is added to the header as a child node and uses the quicklinks slot. Again, like the user account navigation above, each option in the dropdown should be a tf-dropdown-option. However, unlike the user account navigation, this isn't app header specific. Instead it emits the general dropdown event tfDropdownSelected. It is therefore sensible to specifically identify this component with the id property dropdown-id
